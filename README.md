## ✨ المميزات الرئيسية

### 🔐 الأمان والخصوصية
- **نظام PIN محلي** 6 أرقام: فصل كامل بين المستخدمين والمسؤولين
- **تخزين محلي 100%**: جميع البيانات في localStorage - لا تغادر المتصفح
- **بدون Backend**: تطبيق ويب ثابت (Static) بملف HTML واحد فقط (~37 KB)
- **جلسات آمنة**: SessionStorage للتحقق المؤقت

### 👥 إدارة المستخدمين الذكية
- **دورين متكاملين**: مشاهد (عام) و مسؤول (إدارة)
- **إنشاء مستخدمين ديناميكياً**: أضف PINs جديدة برقم إداري (`123987`)
- **إدارة كاملة**: حذف المستخدمين بسهولة
- **تسجيل دخول آمن**: 6 أرقام محلية بدون تشفير خارجي

### 📰 إدارة الأخبار الاحترافية
- **واجهة عامة**: عرض الأخبار للجميع (قراءة فقط)
- **لوحة إدارية**: إضافة/تعديل/حذف الأخبار (للمسؤولين)
- **تصنيفات متقدمة**: الحج، المدينة، باكستان، إيران، أمني، OSINT، عاجل، عام
- **درجات أهمية**: عالي جداً ⚡، مهم ⭐، عادي 📌
- **روابط ومعلومات**: مصدر، رابط خارجي، ملاحظات إضافية، تاريخ الإضافة

### 🎨 واجهة متقدمة (v4.0 جديد!)
- **Dark Mode + Light Mode**: تبديل سلس بين الأوضاع 🌓
- **تخزين التفضيلات**: يحفظ اختيارك تلقائياً
- **ألوان احترافية**: تصميم RTL عربي متطور
- **تأثيرات بصرية**: حركات سلسة وانتقالات جميلة

### 🔍 البحث والفلترة المتقدمة
- **بحث فوري**: ابحث عن الأخبار أثناء الكتابة
- **فلترة حسب التصنيف**: اختر التصنيف المطلوب

### 📚 مكتبة المصادر الكاملة

## 🧪 اختبار محلي + 🚀 نشر Firebase Hosting

### اختبار محلي (نفس مجلد النشر الحقيقي `public/`)

```bash
cd public
python3 -m http.server 8080
# افتح: http://localhost:8080
```

### نشر يدوي إلى Firebase Hosting

```bash
firebase deploy --only hosting
```

### إعداد Firebase Authentication (Google + Email/Password)

1. ملف الإدخال الإنتاجي:
   - `public/index.html`
2. إعدادات Firebase العامة (runtime):
   - `public/firebase-config.js`
3. من Firebase Console فعّل:
   - `Authentication > Sign-in method > Google`
   - `Authentication > Sign-in method > Email/Password`
4. أضف الدومين في:
   - `Authentication > Settings > Authorized domains`

### النشر التلقائي عبر GitHub Actions

- يتم النشر تلقائيًا إلى Firebase Hosting عند `push` إلى الفرع `main`.
- ملف الـ workflow: `.github/workflows/firebase-hosting-merge.yml`
- السر المطلوب في GitHub Actions:
  - `FIREBASE_SERVICE_ACCOUNT_M3TM_RASED`
- مسار النشر الإنتاجي:
  - `firebase.json` → `hosting.public = public`
  - Entry point: `public/index.html`

### فحوصات يدوية بعد النشر (Firebase Console)

- تأكد من تفعيل Email/Password provider.
- تأكد من تفعيل Google provider.
- أضف النطاقات المصرح بها في Authorized domains.
