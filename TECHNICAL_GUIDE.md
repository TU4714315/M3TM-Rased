### مستويات الوصول

```
────────────────────────────────────────────────
│ الدور        │ PINs المتاحة  │ الصلاحيات      │
────────────────────────────────────────────────
│ مسؤول         │ 0000          │ كل شيء        │
│ محرر          │ (ديناميكي)     │ إضافة/حذف     │
│ مستخدم عادي   │ 1234          │ عرض فقط       │
────────────────────────────────────────────────
```

### كيفية عمل التشفير

```javascript
البيانات الأصلية → Base64 Encoding → محفوظ في localStorage
"1234"            "MTIzNA=="        ✅ آمن

// عند التحقق:
معرّف من localStorage → Base64 Decoding → مقارنة مع الإدخال
"MTIzNA=="              "1234"              ✅ تطابق
```

## 🔄 دورة حياة البيانات

### إضافة خبر جديد
```
1. مسؤول يختار: ⚙️ الإدارة
2. يدخل البيانات (عنوان، مصدر، إلخ)
3. يضغط: ➕ إضافة
4. البيانات تُحفظ في localStorage
5. render() تُحدّث الواجهة
6. الخبر يظهر في 📰 الأخبار للجميع
```

### عملية تسجيل الدخول
```
1. المستخدم يدخل PIN (4 أرقام)
2. النظام يبحث في قائمة المستخدمين
3. يقارن PIN المدخل مع PIN المحفوظ (بعد فك التشفير)
4. إذا تطابق: دخول ناجح ✅
5. إذا اختلف: رسالة خطأ ❌
```
